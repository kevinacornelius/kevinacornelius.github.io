Proposal excerpts

If something is easy to use, the more likely that thing will be used. Following this logic, API documentation should make learning and understanding the basic components of an API quick and easy. API documentation must anticipate a developer's needs as illustrated below.




Additionally, when created, documentation should be able to be reused. And as aspects of the API evolve, documentation should be able to be easily updated.

In the illustration below, each numbered box represents a reusable content block. Smaller content blocks (2 and 3) can be reused within different endpoints. Larger content blocks (1) can be combined to execute complex tasks with the API.
